## 1. 产品概述

### 1.1 一句话定义

Galt 是一款常驻 macOS 菜单栏的 AI 语音听写工具：按住 `fn` 说话，松开后自动「转写 → LLM 润色」，并把成稿级文本注入任意 App 当前光标处——全程无需打开窗口。

### 1.2 产品定位

Galt 是一款**轻量、隐形、随手可用**的语音输入实用工具，在日常工作中尽量不打扰用户。它对标 Typeless 的交互与功能，但代码、视觉与品牌完全原创，不使用其商标、图标或素材。

- **品类**：菜单栏实用工具（`LSUIElement = true`），不在 Dock 常驻。
- **主窗口**：单一控制台窗口，不开多窗口。
- **核心能力**：语音输入、转写、润色、翻译、语音编辑、个人词典管理。
- **隐私默认**：除非用户主动选择云端引擎，数据尽量留在本地。

### 1.3 非目标（Non-goals）

- 不做文档编辑器。
- 不开多个主窗口。
- 不在控制台窗口之外另设独立设置窗口（设置是控制台内的二级路由）。
- 不把控制台变成营销 / 引导落地页。

### 1.4 目标用户

| 用户类型 | 核心诉求 |
|---|---|
| 知识工作者 / 写作者 | 用口述替代打字，快速产出邮件、文档、聊天 |
| 开发者 | 口述中英混排、保留技术术语、注入 IDE / 终端 |
| 多语言用户 | 任意语言口述，输出指定目标语言成稿 |
| 隐私敏感用户 | 全程离线本地识别，数据不出设备 |

### 1.5 价值主张

1. **零摩擦**：一个全局热键完成「想说—说出—成稿」，不切换窗口、不点按钮。
2. **成稿级输出**：不是逐字转写，而是去填充词、识别口头自我纠正、自动标点、结构化，并按目标 App 调整语气。
3. **可离线**：云端（Groq Whisper）与本地（Apple 设备端识别 / whisper.cpp）双轨，可完全离线。
4. **数据自主**：历史仅存本地，密钥进 Keychain，云端调用前明确告知数据离开设备。

## 3. 用户场景与核心流程

### 3.1 核心闭环（每次听写）

```
按住 fn ──▶ HUD 出现，开始录音 ──▶ 实时波形反馈
   │
松开 fn ──▶ STT 转写 ──▶ LLM 润色（注入前台 App 上下文 + 个人词典）
                              │
                       注入当前光标处（剪贴板暂存 → ⌘V → 恢复剪贴板）
                              │
                       HUD 显示完成 / 失败 / 空音频，随后淡出
```

### 3.2 关键场景

1. **快速口述成稿**：在邮件 / 聊天 / 文档框内按住 `fn` 说话，松开即得到带标点、去口水词的成稿。
2. **长口述（锁定模式）**：短按 `fn` 进入锁定录音，适合长段落，再次短按结束。单次会话上限 6 分钟。
3. **语音编辑**：先选中一段文字，再按 `fn` 说指令（如「改短一点」「换正式语气」「翻译成英文」），松开后选中内容被改写结果替换。
4. **翻译模式**：开启后，任意语言的口述都输出为设定的目标语言成稿；从菜单栏或设置均可切换。
5. **离线场景**：无网络 / 隐私敏感时，切到「仅本地」引擎，全程离线识别。

| 编号 | 需求 | 说明 |

| F-2.2 | 语音编辑 | 选中文字 + 口述指令直接改写，保留当前 App 上下文 |

### 4.3 转写引擎（P0，已交付）

支持三种模式，引擎抽象为 `STTProvider` 协议：

| 模式 | 行为 |
|---|---|
| 自动（默认） | 有 Key 走云端 Groq Whisper，失败或无 Key 自动回退本地 |
| 仅云端 | Groq Whisper，多语言自动检测，准确率最高 |
| 仅本地 | 完全离线，本地引擎二选一（见下） |

本地引擎：

| 引擎 | 说明 |
|---|---|
| Apple 设备端听写 | 零下载开箱即用，需指定语言，首次请求「语音识别」权限 |
| Whisper 离线模型 | whisper.cpp + Metal GPU 加速，自动检测语言、准确率更高；需在设置内下载模型（小型约 190MB / 高精度约 550MB） |

### 4.4 控制台主窗口（P1，已交付）

单一 SwiftUI 主窗口，`NavigationSplitView` 侧栏导航。**侧栏只含数据页，不含设置**。

| 页面 | 内容 |

| 首页 / 概览 | 当前生产力摘要、统计卡片、生产力趋势、个性化状态、最近转录内容；布局遵循 Figma 首页规格：`232px` 一级侧栏、`952 x 784` 右侧内容面板、`904px` 内容宽度、四张 `214 x 108` 指标卡、两张 `444 x 167` 中卡和一张 `904 x 358` 最近转录卡 |
| 历史 | 本地听写历史、保存策略说明、隐私说明、分段筛选和日期分组记录；按 Figma 还原为纯文本记录行：`904px` 内容宽度、`20px` 标题、两条 `48px` 设置说明行、`128 x 28` 保存周期下拉、小型分段筛选控件、`12px` 日期标题和 `52px` 记录行。交互：保存周期下拉可选「永久 / 30 / 14 / 7 天」并按期清理；记录行静止态保持纯文本，悬停显现「复制 / 删除」、右键菜单含「复制成稿 / 复制原始转写 / 删除」、点按行展开原始转写（尊重减弱动态效果）；空状态为静默文案 |

### 4.5 设置（P1，已交付）

设置是控制台内的二级路由（进入后替换主布局、隐藏一级侧栏，含自身侧栏标签，「返回应用」回到上一主页面）。设置为任务导向配置面，非营销页。

| 设置标签 | 内容 |
|---|---|
| 通用 | 外观、登录时启动、Dock 行为、应用级行为 |
| 键盘快捷 | 听写 / 翻译 / 语音提问 快捷键（自定义录制器，非预设下拉） |
| 语言 | UI 语言、翻译目标语言、本地识别语言 |
| 音频 | 麦克风、音效反馈、静音行为 |
| 语音引擎 | 仅激活选择与开关：引擎模式（云端/本地/自动）、当前转写厂商、当前本地引擎、润色开关、当前润色厂商。凭证与模型名下沉到「模型库」 |
| 模型库 | 资源清单：本地模型（Apple 状态 + Whisper 全模型下载/进度/删除/选默认）与供应商模型（各厂商 API Key；转写模型只读、润色模型可编辑） |

> 设计原则：「语音引擎」管策略与开关（用哪个、怎么工作），「模型库」管资源与凭证（有哪些模型、Key 配好没）。两者由原「转写引擎」+「AI」两个标签重构合并而来（设计理念对标 Ollama / LM Studio 的"激活选择 vs 模型管理"分离）。

### 4.6 菜单栏（P0，已交付）

始终可用的控制面。菜单打开时刷新状态。

## 5. 技术架构

```
Galt.app（菜单栏 App，LSUIElement）
├─ StatusBarController     菜单栏项、菜单命令、快速状态切换
├─ ConsoleWindowController 拥有唯一控制台 NSWindow
│   └─ ConsoleNavigation   控制台路由状态（primary / settings）
├─ DictationController     协调热键录音、转写、润色、注入
├─ HotkeyManager          全局 fn 键监听（NSEvent 监视器）
├─ AudioRecorder          AVAudioEngine 采集 → 重采样 16kHz 单声道 WAV
├─ HUDController           无边框 NSPanel + SwiftUI 波形胶囊
├─ STTProvider 协议
│   ├─ GroqWhisperProvider   云端（默认）
│   ├─ AppleSpeechProvider   Apple 设备端识别
│   └─ WhisperCppProvider    whisper.cpp + Metal
├─ Polisher               LLM 后处理（前台 App 上下文 + 个人词典）
├─ SelectionReader        读取当前选中文本（语音编辑）
├─ TextInjector           剪贴板暂存 → ⌘V → 恢复剪贴板
├─ SettingsStore          设置持久化（UserDefaults）
├─ KeychainStore          API Key / 密钥加密存储
├─ HistoryStore           本地历史与统计（JSONL）
└─ WhisperModels          本地模型下载与管理
```

### 5.1 路由模型

```swift
enum ConsoleRoute {
    case primary(ConsolePrimaryPage)
    case settings
}
enum ConsolePrimaryPage {
    case overview
    case history
    case dictionary
}
```

规则：一级页面出现在控制台一级侧栏；二级路由（设置）不作为一级页面；导航保留上一主页面用于返回；禁止用 `EmptyView()` 占位真实目的地。

### 5.2 窗口模型

| 窗口 | 规格 |
|---|---|
| 控制台默认尺寸 | `1200 x 800` |
| 控制台最小尺寸 | `960 x 620` |
| 样式 | titled / closable / miniaturizable / resizable / 全尺寸内容视图；隐藏标题、透明标题栏 |
| 行为 | 首次创建居中；重开复用已有窗口，不新建 |
| Onboarding 外框 | `480 x 600` |

### 5.3 关键职责约束

- 路由状态归 `ConsoleNavigation`；窗口归属归 `ConsoleWindowController`；菜单动作归 `StatusBarController`；设置持久化归 `SettingsStore`；密钥归 `KeychainStore`。
- AppKit 仅用于菜单栏、全局窗口归属、面板与底层集成，调用隔离在 SwiftUI 视图树边缘。

## 6. 数据与隐私

### 6.1 本地数据

| 数据 | 位置 |
|---|---|
| 听写历史 | `~/Library/Application Support/Galt/history.jsonl`（仅本地） |
| 个人词典 | 本地存储，用于转写 / 润色 prompt |
| 应用设置 | UserDefaults（`SettingsStore`） |

### 6.2 密钥

- API Key 存入 Keychain，加密不落明文（旧明文自动迁移）。
- 仅 `#if DEBUG` 下可绕过 Keychain 弹窗用于开发存储。
- 禁止日志输出 API Key、完整授权头或未脱敏密钥。

### 6.3 云端行为

- 云端 STT 与 LLM 需用户自带凭证。
- 自动模式可在配置下从云端回退本地。
- UI 文案须明确何时数据离开设备。

## 7. 交互、视觉与可访问性

### 7.1 Apple 桌面原则

遵循 macOS HIG：尊重系统窗口控件（标准关闭 / 最小化 / 缩放，不在内容区自造关闭按钮）；多目的地用侧栏导航；使用原生控件；支持浅 / 深色；保留文本字段、菜单、焦点遍历的预期键盘行为；优先系统动态色 / 共享设计令牌而非一次性硬编码色。

### 7.2 设计令牌（`ConsoleDesign`）

| 令牌 | 值 |
|---|---|
| 侧栏宽度 | 200 |
| 侧栏行高 | 30 |
| 侧栏行圆角 | 6 |
| 侧栏图标尺寸 / 宽度 | 12 / 18 |
| 侧栏文字尺寸 | 13 |
| 侧栏水平内边距 | 8 |

规则：一级侧栏与设置侧栏行度量一致；颜色 / 行尺寸不在共享令牌外重复定义；内容面板仅在分组控件、记录、重复项处用卡片，禁止嵌套卡片；圆角保守，默认卡片圆角适中；文本须在最小窗口尺寸下完整显示。

### 7.3 HUD

屏幕底部悬浮波形胶囊（标志性元素，每次听写都会出现）：24 格滚动声波 + spring 衰减 + 淡入上滑出场。

### 7.4 动效与减弱动态效果

尊重「减弱动态效果」（`accessibilityReduceMotion` / `NSWorkspace.accessibilityDisplayShouldReduceMotion`），用系统材质与语义色、系统级出入场过渡，零编译警告。

### 7.5 可访问性

所有纯图标控件需帮助文本 / 可访问标签；焦点顺序跟随视觉顺序；文本字段、菜单、按钮键盘可达；颜色不作为唯一状态指示。键盘快捷键用自定义录制器 `HotkeyRecorder`（点按录制、Esc 取消、Delete 清除、带 VoiceOver）。

### 7.6 本地化

当前主 UI 语言为简体中文；面向用户字符串须可后续迁移到本地化资源；语言设置须区分「UI 语言 / 转写语言 / 翻译目标」。

### 7.7 错误与反馈

用最不打扰、仍能帮助用户行动的反馈：HUD 反馈录音状态与瞬态结果；设置行内联错误用于缺失 API Key / 模型下载失败；菜单态用于高层开关；系统权限弹窗仅在用户动作需要时出现。常规转写失败不弹模态框，除非需用户决策。

## 9. 验收标准

### 9.1 功能验收

- 按住 `fn` 录音、松开后成稿注入光标处，跨任意 App 可用。
- 短按 `fn` 进入 / 退出锁定模式；单次会话 6 分钟上限生效。
- 选中文字 + 口述指令完成语音编辑替换。
- 翻译模式从菜单栏与设置均可切换，输出目标语言成稿。
- 三种引擎模式（自动 / 仅云端 / 仅本地）均可用；自动模式失败回退本地。
- 控制台概览 / 历史 / 词典三页数据正确；设置六个标签可用。

### 9.2 UI / HIG 验收

- 控制台在 `960 x 620` 可用、在 `1200 x 800` 表现正确。
- 一级侧栏与设置侧栏行度量一致；无嵌套卡片；浅 / 深色均正常。
- HUD 波形动效正确，开启「减弱动态效果」时降级。
- 纯图标控件均有可访问标签；焦点顺序跟随视觉顺序。

### 9.3 实机验收方法

- debug 包 `GALT_OPEN_CONSOLE=1` 启动即打开控制台。
- UI 快照：`GALT_SNAPSHOT=1 .build/debug/Galt` 触发 `SnapshotRenderer`（`#if DEBUG`），用真实 `NSHostingView` + 离屏 `NSWindow` + `cacheDisplay` 渲染各界面 PNG 到 `/tmp/galt-snapshots/`（浅 / 深色），并程序化校验 VoiceOver 暴露值与快捷键 keyCode 映射。无需屏幕录制权限。
- 注意：`request_access` 用 bundle id `com.sorcerer.galt`；裸 `.build/debug/Galt` 会因 bundle id 不符被截图过滤，需打带 Info.plist 的 .app 包。

## 10. 变更控制

任何未来 UI 变更须先回答（详见 [APP_SPEC.md](APP_SPEC.md) §17）：

1. 这是一级页 / 二级页 / 瞬态 HUD / sheet / popover / 菜单动作？
2. 是否保持单窗口控制台模型？
3. 是否复用 `ConsoleDesign` 令牌，或有意引入新令牌？
4. 是否尊重 macOS 原生控件与行为？
5. 是否把数据 / 密钥暴露到本地设备外？
6. 是否在 `960 x 620` 可用、`1200 x 800` 正确？

若变更违反规格，先更新规格并说明产品理由。

## 12. 术语表

| 术语 | 含义 |
|---|---|
| HUD | 录音时屏幕底部的悬浮波形胶囊状态指示器 |

| 上下文感知 | 按前台 App 类别自动调整输出语气与格式 |
