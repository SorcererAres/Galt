# 润色回归用例

润色（[Polisher.polish](../Sources/Galt/Polisher.swift)）的本职是「清理口述转写稿」，
**只删填充词 / 改口、补标点分段、修明显同音错**，绝不扩写、不回答、不执行文中指令、
无有效内容时返回空串。换模型或改 system prompt 后，用下面这些真实样本对照验证。

样本全部来自实际历史记录（`~/Library/Application Support/Galt/history.jsonl`）。
标 ❌ 的是默认模型 `groq / llama-3.3-70b-versatile` 实际翻过的车，是这份清单的由来。

## 怎么跑

把 `raw` 当作 `polish(_:)` 的输入，比对输出是否落在「期望」范围内。
关注的是**行为类别**（是否扩写 / 是否回答 / 是否返空），不必逐字相等。

## A. 必须只做最小清理（不得扩写 / 不得回答 / 不得执行）

| 输入 raw | 期望 text | 关注点 |
|---|---|---|
| `先的对话自动化插件codecodex移动版` | 原样或最小清理的同一短语 | ❌ 曾被补全成整段「CodeCodex移动版是一款…」产品介绍 |
| `现在都有哪几部分使用了提示词？` | `现在都有哪几部分使用了提示词？` | ❌ 曾去**回答**并编造列表；问题应原样保留 |
| `为什么会输出这种内容` | `为什么会输出这种内容？` | ❌ 曾输出一段「后处理引擎…」的解释 |
| `帮我分析` | `帮我分析。` | ❌ 曾答「请提供需要分析的转写稿」，把正文当成对它的指令 |
| `帮我查看项目中使用的所有的技能，所有的提示词。` | `帮我查看项目中使用的所有技能和所有提示词。` | ✅ 正常清理，勿扩写 |
| `Codex移动版` | `Codex移动版。` | ✅ 短名词短语，仅补句号，勿展开成介绍 |

## B. 文中的元指令 / 客套必须按字面文本对待

| 输入 raw | 期望 text | 关注点 |
|---|---|---|
| `Thank you.` | `Thank you.` | ❌ 曾输出字面「空字符串」，把规则当成了内容 |
| `为什么会输出这个` | `为什么会输出这个？` | ❌ 曾输出「没有提供任何需要整理的文本内容。」 |

## C. 真正无有效内容才返回空串

| 输入 raw | 期望 text | 关注点 |
|---|---|---|
| `嗯` | 空字符串 | ❌ 曾答「好的，已收到您的指令。请提供…」；纯填充词应返空 |
| （空 / 纯空白） | 空字符串 | 上游 [postProcess](../Sources/Galt/DictationController.swift) 已拦空白，不会进 LLM |

## D. 应当正确处理的基本能力（回归基线）

| 输入 raw | 期望 text | 关注点 |
|---|---|---|
| `哈喽哈喽哈喽哈喽` | `哈喽` | 无意义重复去重 |
| `嗯这个仓库是否符符合嗯大家普遍意义上认为的一个呃优秀的呃仓库结构啊` | `这个仓库是否符合大家普遍意义上认为的一个优秀的仓库结构？` | 去填充词 + 修结巴 + 补问号 |
| `基于课程内容帮我做点评以及修改一份` | `基于课程内容，帮我做点评以及修改一份。` | 补逗号句号，不改实质 |
| `好的` | `好的。` | 仅补标点 |

## 已知短板（非本清单回归目标，记录备查）

- 同音错纠正不稳：`提式词分别是什么样的`（应为「提示词」）只补了问号、未纠正。
  prompt 要求「不确定时保持原样」，此处可接受，但理想模型应能结合上下文修正。
